#commits_function: 
Crear un archivo markdown con los ultimos cambios realizados en el codigo

##where or when use it:
Usar cuando se necesite crear un archivo markdown con los ultimos cambios realizados en el codigo

##how to use it:
1. Identificar los cambios realizados en el codigo y colocarle un titulo breve a cada cambio
2. Crear un nuevo archivo markdown en la carpeta Verification con los ultimos cambios realizados en el codigo y una explicacion detallada de los cambios realizados 
3. Tener en cuenta preguntas como: 
- ¿Qué cambios se realizaron?
- ¿Por qué se realizaron estos cambios?
- ¿Cómo se realizaron estos cambios?
- ¿Qué impacto tienen estos cambios en el sistema?
- ¿Qué hace/n la/s funcion/es?
- ¿que buenas practicas siguio?
- ¿Que estructura de carpetas tuvo la modificacion? por ejemplo, "cambios en app/http/controladorCargo y routes/api.php"
4. Verificar que los cambios estén correctamente documentados y que el archivo markdown tenga un formato limpio y ordenado
